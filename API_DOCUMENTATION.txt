MARKETPLACE API DOCUMENTATION
================================================================================
This document provides a detailed overview of all API endpoints, including
request parameters, request bodies (DTOs), and response structures.

--------------------------------------------------------------------------------
1. AUTHENTICATION & USER MANAGEMENT (UserController)
--------------------------------------------------------------------------------
Base URL: /api/v1/users

1.1 Register User
    Authentication: Public
    POST /register
    Consumes: multipart/form-data
    Request Parts:
      - data (UserRegistrationRequest): JSON string
        - fullName (String): User's full name.
        - email (String): User's email address.
        - password (String): User's password (min 8 characters).
        - phoneNumber (String): User's phone number.
        - bio (String, optional): A short biography.
        - location (String, optional): User's location.
      - profilePic (MultipartFile): Image file for the profile picture.
    Response: UserRegistrationResponse
      - id (UUID): The unique identifier for the user.
      - fullName (String): The user's full name.
      - email (String): The user's email address.
      - phoneNumber (String): The user's phone number.
      - location (String): The user's location.
      - role (Role): The role assigned to the user (e.g., USER).
      - createdAt (LocalDateTime): The timestamp when the user was created.

1.2 Login User
    Authentication: Public
    POST /login
    Consumes: application/json
    Request Body: UserLoginRequest
      - email (String): User's email address.
      - password (String): User's password.
    Response: UserLoginResponse
      - id (UUID): The unique identifier for the user.
      - email (String): The user's email address.
      - role (Role): The role of the user.
      - accessToken (String): JWT token for authenticating subsequent requests.
      - refreshToken (String): Token to refresh the access token.

1.3 Get My Profile
    Authentication: Authenticated (User)
    GET /me
    Headers: Authorization: Bearer <token>
    Response: UserDetailsResponse
      - id (UUID): The user's unique identifier.
      - fullName (String): The user's full name.
      - email (String): The user's email address.
      - phoneNumber (String): The user's phone number.
      - location (String): The user's location.
      - bio (String): The user's biography.
      - role (String): The user's role.
      - profilePic (ProfilePicResponse): The user's profile picture details.
        - id (Long): The ID of the profile picture.
        - imageUrl (String): The URL of the profile picture.
        - imagePublicId (String): The public ID for the image in the cloud storage.
      - createdAt (LocalDateTime): The timestamp when the user was created.
      - updatedAt (LocalDateTime): The timestamp when the user was last updated.

1.4 Get Profile Picture
    Authentication: Authenticated (User)
    GET /me/profile-pic
    Headers: Authorization: Bearer <token>
    Response: ProfilePicResponse
      - id (Long): The ID of the profile picture.
      - imageUrl (String): The URL of the profile picture.
      - imagePublicId (String): The public ID for the image in the cloud storage.

1.5 Update Profile
    Authentication: Authenticated (User)
    PUT /me
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - data (UserUpdateRequest): JSON string
        - fullName (String): User's full name.
        - phoneNumber (String): User's phone number.
        - location (String): User's location.
        - bio (String): User's biography.
      - profilePic (MultipartFile): Image file for the profile picture.
    Response: UserUpdateResponse
      - id (UUID): The user's unique identifier.
      - fullName (String): The user's full name.
      - email (String): The user's email address.
      - phoneNumber (String): The user's phone number.
      - location (String): The user's location.
      - bio (String): The user's biography.
      - updatedAt (LocalDateTime): The timestamp when the user was last updated.

2. SELLER MANAGEMENT (SellerController)
--------------------------------------------------------------------------------
Base URL: /api/v1/sellers

2.1 Register Seller
    Authentication: Public
    POST /register
    Consumes: multipart/form-data
    Request Parts:
      - data (SellerRegistrationRequest): JSON string
        - fullName (String): Seller's full name.
        - email (String): Seller's email address.
        - password (String): Seller's password (min 8 characters).
        - phoneNumber (String): Seller's phone number (Ghana format).
        - storeName (String): The name of the seller's store.
        - businessAddress (String): The business address of the seller.
        - taxId (String): The seller's tax identification number.
        - location (String, optional): The seller's location.
        - sellerBio (String, optional): A short biography for the seller.
        - storeDescription (String, optional): A description of the store.
      - profilePic (MultipartFile): Image file for the seller's profile picture.
    Response: SellerRegistrationResponse
      - id (UUID): The unique identifier for the seller.
      - fullName (String): The seller's full name.
      - email (String): The seller's email address.
      - phoneNumber (String): The seller's phone number.
      - storeName (String): The name of the seller's store.
      - businessAddress (String): The business address of the seller.
      - role (String): The role assigned to the seller (e.g., SELLER).
      - createdAt (LocalDateTime): The timestamp when the seller was created.

2.2 Login Seller
    Authentication: Public
    POST /login
    Consumes: application/json
    Request Body: SellerLoginRequest
      - email (String): Seller's email address.
      - password (String): Seller's password.
    Response: SellerLoginResponse
      - id (UUID): The unique identifier for the seller.
      - fullName (String): The seller's full name.
      - email (String): The seller's email address.
      - storeName (String): The name of the seller's store.
      - role (Role): The role of the seller.
      - accessToken (String): JWT token for authenticating subsequent requests.
      - refreshToken (String): Token to refresh the access token.

2.3 Get My Profile (Seller)
    Authentication: Authenticated (Seller)
    GET /me
    Headers: Authorization: Bearer <token>
    Response: SellerDetailsResponse
      - id (UUID): The seller's unique identifier.
      - fullName (String): The seller's full name.
      - email (String): The seller's email address.
      - phoneNumber (String): The seller's phone number.
      - location (String): The seller's location.
      - sellerBio (String): The seller's biography.
      - storeName (String): The name of the seller's store.
      - storeDescription (String): The description of the store.
      - businessAddress (String): The business address of the seller.
      - taxId (String): The seller's tax identification number.
      - isVerified (boolean): Whether the seller is verified.
      - role (String): The seller's role.
      - profilePic (ProfilePicResponse): The seller's profile picture details.
        - id (Long): The ID of the profile picture.
        - imageUrl (String): The URL of the profile picture.
        - imagePublicId (String): The public ID for the image in the cloud storage.
      - createdAt (LocalDateTime): The timestamp when the seller was created.
      - updatedAt (LocalDateTime): The timestamp when the seller was last updated.

2.4 Get Seller by ID (Public)
    Authentication: Public
    GET /{sellerId}
    Response: SellerDetailsResponse
      - (Same as 2.3 Get My Profile (Seller))

2.5 Update Profile (Seller)
    Authentication: Authenticated (Seller)
    PUT /me
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - data (SellerUpdateRequest): JSON string
      - profilePic (MultipartFile): Image file
    Response: SellerUpdateResponse
      - id (UUID): The seller's unique identifier.
      - fullName (String): The seller's full name.
      - email (String): The seller's email address.

3. PRODUCT MANAGEMENT - SELLER (SellerProductController)
--------------------------------------------------------------------------------
Base URL: /api/v1/seller

3.1 Create Category
    Authentication: Authenticated (Seller)
    POST /categories
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - data (CreateCategoryRequest): JSON string
        - name (String): The name of the category.
      - icon (MultipartFile): Image file for the category icon (optional).
    Response: CategoryResponse
      - id (UUID): The unique identifier for the category.
      - name (String): The name of the category.
      - slug (String): The slug for the category URL.
      - icon (CategoryIconResponse): The category icon details.
        - id (Long): The ID of the icon.
        - imageUrl (String): The URL of the icon.
        - publicId (String): The public ID for the icon in the cloud storage.

3.2 Update Category
    Authentication: Authenticated (Seller)
    PUT /categories/{categoryId}
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - data (CreateCategoryRequest): JSON string
        - name (String): The new name of the category.
      - icon (MultipartFile): New image file for the category icon (optional).
    Response: CategoryResponse
      - (Same as 3.1 Create Category)

3.3 Delete Category
    Authentication: Authenticated (Seller)
    DELETE /categories/{categoryId}
    Headers: Authorization: Bearer <token>
    Response: String (message)

3.4 Get Seller Categories
    Authentication: Authenticated (Seller)
    GET /categories
    Headers: Authorization: Bearer <token>
    Response: List<CategoryResponse>
      - (A list of CategoryResponse objects as defined in 3.1)

3.5 Add Product
    Authentication: Authenticated (Seller)
    POST /products
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - data (CreateProductRequest): JSON string
        - name (String): The name of the product.
        - description (String): The description of the product.
        - price (BigDecimal): The price of the product.
        - categoryId (UUID): The ID of the category this product belongs to.
        - stock (int): The available stock for the product.
        - brand (String, optional): The brand of the product.
        - discount (BigDecimal, optional): The discount percentage for the product.
      - images (List<MultipartFile>): Image files for the product (optional).
    Response: CreateProductResponse
      - id (UUID): The unique identifier for the product.
      - name (String): The name of the product.
      - description (String): The description of the product.
      - price (BigDecimal): The price of the product.
      - categoryName (String): The name of the product's category.
      - stock (int): The available stock for the product.
      - brand (String): The brand of the product.
      - discount (BigDecimal): The discount percentage for the product.
      - status (ProductStatus): The status of the product (e.g., ACTIVE).
      - images (List<ProductImageResponse>): A list of product images.
        - id (Long): The ID of the image.
        - imageUrl (String): The URL of the image.
        - publicId (String): The public ID for the image in the cloud storage.

3.6 Update Product
    Authentication: Authenticated (Seller)
    PUT /products/{productId}
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - data (UpdateProductRequest): JSON string
        - name (String): The new name of the product.
        - description (String): The new description of the product.
        - price (BigDecimal): The new price of the product.
        - categoryId (UUID): The new category ID for the product.
        - brand (String): The new brand of the product.
        - discount (BigDecimal): The new discount percentage for the product.
      - images (List<MultipartFile>): New image files for the product (optional).
    Response: CreateProductResponse
      - (Same as 3.5 Add Product)

3.7 Replace All Product Images
    Authentication: Authenticated (Seller)
    PUT /products/{productId}/images/replace
    Headers: Authorization: Bearer <token>
    Consumes: multipart/form-data
    Request Parts:
      - images (List<MultipartFile>): New image files for the product.
    Response: CreateProductResponse
      - (Same as 3.5 Add Product)

3.8 Update Stock

      - (Same as 3.5 Add Product)
